## About Me

I build systems that give AI persistent memory and personal identity. My work focuses on unifying fragmented user data into coherent long-term knowledge graphs that AI agents can retrieve, reason over, and personalize against.

I am strongly convinced that in the agentic economy everyone will have an **AI Identity** they can import without re-introducing themselves to every new vertical tool. I’m stubbornly pursuing this vision as the founder of **Enagrams**.

Enagrams is an AI memory and identity infrastructure layer that integrates with a user’s digital footprint (Google, Slack, Notion, browser activity, LLM transcripts, files, events) and transforms it into a structured, queryable knowledge graph. Developers use Enagrams to build AI agents that already understand their users.

I am also a **VC Fellow at NEA** and a **full-ride scholarship recipient at Columbia University**.

Alongside any public contributions, this GitHub serves as a space for research distillations.

## Focus Areas

### AI Memory and Identity Systems

Designing and implementing long-horizon memory architectures that combine:

- **Graph-structured identity representation**  
- **Temporal weighting** of preferences and behaviors  
- **Retrieval pipelines** blending GraphRAG and embedding-based search  
- **Multi-source ingestion pipelines** with normalization and entity resolution  
- **Real-time context synchronization** across SaaS, local, and browser environments  

My goal is to make user memory **as portable and interoperable as financial identity**.  
AI products should not start from zero context every time a session begins.
